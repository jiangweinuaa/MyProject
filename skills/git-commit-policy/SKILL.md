# Git 提交策略技能

## 技能名称
git-commit-policy

## 适用范围
当 AI 准备提交代码到 Git 仓库时，必须先询问用户确认。

## 核心原则
1. **禁止自动提交未经测试的代码**
2. **提交前必须询问用户**
3. **确保代码已测试通过**

## 操作流程

### 步骤 1：检查代码状态
```bash
cd /home/admin/.openclaw/workspace && git status
```

### 步骤 2：询问用户（必须执行）
在提交代码前，必须向用户发送以下确认消息：

```
📝 检测到有未提交的代码变更，是否提交到 Git？

变更文件：
[列出主要变更文件]

提交信息：
[建议的提交信息]

请确认：
1. 是，提交代码（已测试通过）✅
2. 否，暂不提交 ❌
3. 修改提交信息 ✏️
```

### 步骤 3：根据用户回复执行

**用户回复"是"或"1"或"✅"**：
```bash
git add -A
git commit -m "提交信息"
git push origin master
```

**用户回复"否"或"2"或"❌"**：
- 不执行提交操作
- 回复用户："好的，代码暂不提交"

**用户回复"3"或"✏️"**：
- 询问用户新的提交信息
- 使用用户提供的提交信息执行提交

## 禁止行为
❌ 未经用户确认自动提交代码
❌ 提交未经测试的代码
❌ 跳过询问步骤直接 push

## 测试要求

在询问用户之前，确保：
1. ✅ 前端编译通过
2. ✅ 后端编译通过
3. ✅ 服务运行正常
4. ✅ 核心功能测试通过

如果任何一项失败，在询问时注明：
```
⚠️ 注意：部分测试未通过
- [ ] 前端编译
- [ ] 后端编译
- [ ] 服务运行
- [ ] 功能测试

建议先修复问题再提交。
```
